## UI 필드 정의

### 리스트 카드 (각 축의 항목)
- 이름 + 관계 (예: "김철수 · 친구/지인")
- 금액 (serif, 콤마 포맷: "100,000원")
- 일시 (상대 시간 또는 날짜)
- 참석 뱃지 (참석/불참)

### 상세 드로어
- 이름
- 금액 (콤마 포맷 + "원")
- 관계 (카테고리 + 상세)
- 축의 방식
- 참석 여부
- 일시
- 하단: 수정 / 삭제 버튼

### 요약 카드
- 총 축의금 (콤마 포맷)
- 축의 건수
- 참석 수

### CSV 칼럼
이름, 관계, 관계상세, 금액, 축의방식, 참석여부, 일시

## 입력 폼 정의

### 필드 목록

| 필드 | 타입 | 필수 | 기본값 | 비고 |
|------|------|------|--------|------|
| 이름 | text | O | — | — |
| 금액 | number | O | 0 | 천 단위 콤마 포맷, "원" 접미사 |
| 관계 | select | X | 미선택 | 가족/친척, 친구/지인, 동문/동창, 직장동료, 스승/제자, 기타모임 |
| 관계 상세 | text | X | — | 최대 40자 (예: "고등학교 동창") |
| 축의 방식 | select | X | 미선택 | 간편송금(토스), 간편송금(카카오페이), 계좌이체, 현금 |
| 참석 여부 | checkbox | X | 미체크 | — |

## 결정 사항

- wedding 단위 단일 장부, side 필터 없음 (v3는 채널 분리 없이 host 슬롯 구조)
- 감사 인사 기능 없음
- 시간순 고정, 검색/정렬 UI 없음
- 축의 방식 4종: 간편송금(토스), 간편송금(카카오페이), 계좌이체, 현금
- 무한 스크롤 (cursor 기반 페이지네이션)
- 수정: 모든 필드 수정 가능 / 삭제 가능 / 중복 허용
- 관계 카테고리 6종: 가족/친척, 친구/지인, 동문/동창, 직장동료, 스승/제자, 기타모임 (guest-web 기준)
- 금액: 천 단위 콤마 포맷 + "원" 표시
- Permission: Host 전체 (groom, bride, groom_father, groom_mother, bride_father, bride_mother)
- Navigation: MyWeddingPage 모청 카드 → `/wedding/{weddingId}/report`
