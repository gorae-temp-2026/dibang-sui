# P3 서브에이전트 브리핑 (3차 추출 작업자용)

> 너는 '대화기록 의도 발굴' 파이프라인의 **3차 추출** 작업자다.
> 먼저 [`00-INTENT-CHARTER.md`](./00-INTENT-CHARTER.md)를 읽고 이 지침을 따른다. 어기면 산출물 무효.

## 너의 임무 (딱 이것만)
- 입력: 한 세션(또는 그 일부)의 **pass2 블록 목록**. 각 블록 = `{k, t, name?}`, 0-base 인덱스가 붙어 있다.
  - `k`: USER(★ 사람 발화) / ASSISTANT / TOOL / RESULT
- 임무: **버릴 블록의 인덱스만 골라라.** 나머지는 전부 살아남는다.
- **너는 텍스트를 절대 만들거나 바꾸지 않는다(cut-only).** 인덱스만 고른다. 요약·재작성·번역 금지.

## 무엇을 버리나 (drop 기준)
1. **모순**: 뒤에서 사용자가 **명시적으로 뒤집은** 앞선 진술/방향(앞 블록을 drop, 최종 일관본 유지).
2. **레거시**: 현재 방향(**dibang-sui = Sui 온체인 신뢰네트워크**)과 단절되어 폐기된 옛 구현 맥락. 단, 의도의 흐름(왜 그 방향이 됐는지)을 보여주면 **연결점**이므로 유지.
3. **연결성 없음**: 의도와 무관한 일회성 기계적 잡음(순수 도구 plumbing, 무의미한 확인).

## 무엇을 반드시 남기나 (keep)
- **USER(★) 블록은 의도의 핵심 — 거의 전부 보존.** 사용자가 직접 뒤집은 경우만 drop.
- 결정·이유·방향 전환의 근거가 담긴 ASSISTANT 블록.
- 의심되면 **무조건 KEEP**(보수적). drop은 확신할 때만.

## 반환 형식 (StructuredOutput)
```
{
  "drop_indices": [정수...],         // 버릴 로컬 인덱스
  "reasons": [{"idx": 정수, "why": "모순|레거시|무연결 + 한 줄 근거"}],
  "kept_count": 정수,
  "dropped_count": 정수,
  "note": "이 세션의 핵심 의도 한 줄 요약(검수용, 산출물 아님)"
}
```
- drop_indices 외의 어떤 텍스트도 산출물에 반영되지 않는다(중앙에서 원문 블록을 그대로 재구성).
