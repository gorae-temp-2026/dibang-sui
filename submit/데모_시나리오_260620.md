2. **디방인연** — 스와이프·이음·대화·**선물**(관계 행동 = 신뢰 신호 적립).
3. **웨딩 라운지 / 모이가모인곳 광장** — 하객 모이들(손그림), 꾸미기·인테리어.
4. **모이 클릭 → 공유 프로필** — ① 인연 연결 · ② 우리 signal · ③ 익명 신뢰범위 + 사진·후크.

6. **Reveal** — 가상 소비자/프로토콜이 **'철수가 누군지 모른 채'** Moi Credit을 read. → 슬로건 **"관계가 전부다 / 누군지 몰라도 신용을 안다."**

## 4. thesis 확장 — "내 결혼식에 누가 올까" → DeFi (Sui 네이티브 아키텍처 티저)
- Moi Credit·신뢰네트워크로 **철수♥영희 결혼식의 예상 하객 예측** (누가 올지 = 호혜 관계·이음 강도 기반).
- 예상 하객 → **예상 부조(축의금) = 예측 가능한 미래 현금 유입**.
- **DeFi 연결:** 사회적 신용 + 예상 부조를 근거로 한 **무담보 "웨딩 대출"** (미래 부조 = 현금흐름 담보).

### 4-1. Sui 네이티브 연결 (조사 결과, 2026-06)
- **Moi Credit = Move 오브젝트(key+store) = composable 신용 오브젝트.** 다른 프로토콜이 read(우리 "정체 모른 채 read" thesis).
- **Navi Protocol** (Sui 대표 랜딩) — 기본은 과담보지만, ① **Isolated Markets**(신규·리스크 자산 격리 상장, 2026) ② **온체인 신용점수 → 오라클 → 무담보 대출** 패턴(예: DeAgent AI 오라클)이 존재. → **Moi Credit을 평판 오라클 입력으로 Navi가 read** → 사회신용 기반 웨딩 대출.
- **DeepBook** (Sui 네이티브 CLOB·유동성 엔진, ~390ms 정산) — 대출 자산·요네↔스테이블 정산·청산의 **유동성 레일**. Navi도 청산에 DeepBook 사용.
- **그림:** Moi Credit(평판) → Navi(오라클 read·무담보 대출) → DeepBook(유동성·정산).

### 4-2. 데모 범위
- **티저까지만:** ① Moi Credit 온체인 오브젝트 ② 가상 소비자/프로토콜이 익명 read ③ "하객 예측 → 웨딩 대출(Navi·DeepBook)" 아키텍처 카피/슬라이드.
- **실제 Navi/DeepBook 대출 연동·실금융행위 = 로드맵**(Navi 무담보는 오라클·리스크심사 필요, D-day 불가). 데모는 개념·read 시연.
- **'우리 결혼식, 누가 올까' 카드(My event)는 제품 기능이 아니라 데모 영상 전용 시연 요소**다. 프로덕트 UI(MyWeddingPage)에선 **주석처리로 비활성**(코드 보존), 촬영 시에만 임시 활성화한다(T14, `DefiTeaserCard`).
