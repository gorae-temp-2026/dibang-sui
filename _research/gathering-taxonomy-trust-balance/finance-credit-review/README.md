# 금융·신용 시스템 대비 검토 (finance-credit-review)

> gathering-taxonomy-trust-balance 하위 리서치. PHI-5(전이/합성)로 들어가기 **전에**, 지금까지 세운 신뢰→신용 모델이 기존 금융·신용 시스템의 맥락과 잘 어우러지는지 점검한다. (2026-06-17 착수)

## 목적
1. **정합** — 우리 모델(1~4층, PHI-1·PHI-4 결정)이 기존 금융·신용의 논리와 잘 어우러지나.
2. **엇나감 점검** — 기존을 따라할 필요는 없지만, 논리적으로 너무 엇나간(설명 안 되는) 부분은 없나.
3. **메울 갭** — 기존 시스템이 못 하는, 우리가 메울 수 있는/메워야 하는 차별화 지점은 어디인가.

## 접근 — 매핑 리뷰 우선
각 도메인의 핵심 개념을 우리 4층에 대응시키는 **매핑표**를 만들고 정합/엇나감/갭을 플래그한다. 도메인별 딥리서치는 갭으로 지목된 곳만 후행한다(빠른 전체 그림 우선).

**절차 — 도메인마다 3단계로 확실하게:**
1. **리서치** — 그 시스템의 핵심 개념·작동 메커니즘·용어·대표 사례를 *출처와 함께* 수집한다(우리 모델 대조는 아직 안 함, 순수 수집). → 도메인 파일 §1 리서치.
2. **매핑** — 수집한 개념을 우리 4층 어디에 대응되는지 표로 작성한다(판정 없이 위치만). → §2 매핑.
3. **검토** — 각 대응을 정합(○/△/✗)·엇나감(논리 어긋남·리스크)·우리가 메울 갭으로 판정한다. → §3 검토.

이후 4개 도메인 §3을 모아 SUMMARY(종합) + 어드버서리얼 검증.

**매핑표 컬럼(공통, §2·§3에서 채움):**

| 기존 개념 | 한 줄 정의 | 우리 4층 대응 위치 | 정합(○/△/✗) | 엇나감·리스크 | 우리가 메울 갭 |
|---|---|---|---|---|---|

## 우리 모델 기준점 (대응 대상)
- **1층 분류** — 이벤트·집단·관계 택소노미(코어 17축·R축).
- **2층 fold** — EM 호혜잔액 = (ℤ,+) 아벨군·반대칭 net(청산) / CS 유대 = 방향별 (ℤ,+) 군(무청산). (07 §1, §1-1)
- **3층 Φ** — 유방향·가중 재귀 전파(상대 신용으로 가중, 행위 A→B·신용 영향 B→A) + 이행 = node 신뢰도가 EM 노출 엣지로 리스크 확산. (09 §1-1)
- **4층 금융** — 신용 → 대출·보증(미구축).
