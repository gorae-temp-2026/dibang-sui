# 02. 축 간 관계 — 중복·포함·상관·직교성

> 선행 문서: [01-axes.md](./01-axes.md). 여기서는 정제된 축들 사이의 관계를 4종류(중복/포함/상관/직교)로 구분하고, 최종 "직교 코어 축 집합"을 확정한다.

---

## 0. 관계의 4종류 정의

| 관계 | 정의 | 처리 |
|---|---|---|
| **중복** | 두 축이 사실상 같은 것을 측정 (값이 거의 1:1 대응) | 하나로 합침 |
| **포함(함의)** | 축 X의 특정 값이 축 Y의 값을 논리적으로 강제 (X=x ⇒ Y=y) | 둘 다 유지하되 함의 규칙을 명시 (검증 규칙으로 활용) |
| **상관** | 경험적으로 같이 움직이지만 반례 존재 | 둘 다 유지, 반례를 기록 |
| **직교** | 서로 정보를 주지 않음 | 코어 축 후보 |

---

## 1. 중복 — 합치는 축 4쌍

### 1-1. 가고 싶은/가기 싫은 ↔ 참여 강제성(N1)
"가기 싫은데 가는 모임"의 원인 대부분이 의무·강제다. 다만 완전한 중복은 아니다 — 자발 모임도 가기 싫을 수 있다(귀찮은 동호회 정모). **결정: 객관 축으로는 N1(자발/의무/강제)을 쓰고, 원본 축은 B층위(참가자별 함수)로 강등해 보존.** 이렇게 하면 "강제성은 낮은데 참여 욕구도 낮음 = 관계가 식고 있다는 신호" 같은 두 축의 차이 자체가 정보가 된다.

### 1-2. 의도한/의도하지 않은 ↔ 초점성(N2, Goffman)
원본 의도 축은 "사전 계획"을 묻지만, 실제로 가르고 싶었던 것은 "엘리베이터에서 우연히 같이 있는 것도 모임인가"였다. Goffman의 구분이 이를 더 정밀하게 처리한다: 엘리베이터 동승은 **unfocused gathering**(공존하지만 공동 초점 없음), 거기서 대화가 시작되면 **focused encounter**로 전이. **결정: N2로 대체.** 의도(계획) 여부는 N2와 독립적으로 남는 정보가 거의 없음 — 계획된 unfocused(각자 독서실에 옴), 비계획 focused(우연히 만나 술자리) 모두 N2가 본질을 잡는다.

### 1-3. 혼성/동성 ↔ 구성원 동질성(N7)
성별 동질성은 동질성의 한 특수 사례. 학연·지연·혈연·직연·취향이 같은 자리에 있어야 할 형제 축. **결정: N7로 흡수.** 단, 소개팅·미팅·데이트처럼 성별 구성이 모임의 정의 자체인 경우가 있으므로 N7의 값 목록에 "성별"을 유지.

### 1-4. 무료/유료 ↔ 금전 흐름 구조(N6)
유/무료는 N6의 "없음 vs 나머지" 투영(projection). **결정: N6으로 흡수.** N6이 더 많은 정보를 담는다: 회비(동호회), 일방 지불(콘서트), 부조(결혼식), 쌍방 교환(중고거래), 임금(회사 — 역방향 지불!)은 전부 "유료"지만 금융적으로 전혀 다른 구조다. 특히 회사는 참가자가 돈을 받는 모임이라는 점에서 유/무료 이분법이 완전히 실패하는 사례.

## 2. 포함(함의) — 논리 규칙 7개

검증 규칙으로 쓸 수 있는 단방향 함의들. (⇒의 역방향은 성립하지 않음에 주의)

| # | 규칙 | 근거 | 역방향 반례 |
|---|---|---|---|
| F1 | 이벤트이다 ⇒ 기한 있음 | 모든 이벤트는 시작·끝이 있음. 따라서 **기한 축은 집단에만 유의미한 축** | 기한 있는 것이 모두 이벤트는 아님 (고3 학급 = 기한 있는 집단) |
| F2 | **통과의례**(N11=통과) ⇒ focused(N2) | 통과의례는 정의상 공동 주의 초점(신랑신부, 고인)을 요구 (Collins의 mutual focus). ※ 매핑 검증에서 "정기 의례"까지 확장하면 반례 발생(축제·선거 = 정기적이지만 unfocused 장) → 통과의례로 한정 | focused라고 의례는 아님 (회의) |
| F3 | 부조형 금전 흐름(N6=부조) ⇒ 호스트-게스트 구조(N4) | 부조는 게스트→호스트 방향이 정의에 포함됨 | 호스트 구조라고 부조가 있진 않음 (집들이도 선물 관습은 있으나 파티 일반은 아님) |
| F4 | 일방향(축12=1:N) ⇒ 비대칭 구조(N4≠대칭) | 방송형 흐름은 송신자/수신자 역할 분화를 전제 | 비대칭이어도 다방향일 수 있음 (회사 회의: 위계 있지만 N:N) |
| F5 〔v2 수정〕 | unfocused(N2) ⇒ S2≠호스트(중심 인물 없음) | 공동 초점이 없는데 모임의 중심 인물이 있을 수 없음 (버스 기사는 호스트가 아니라 공급자). ※ v1은 후건이 버린 축 '모임장'을 가리켰음 — 모임장이 S2로 흡수되어 현 축으로 재서술 | 중심 인물 없어도 focused 가능 (친구 술자리) |
| F6 | 귀속 멤버십(N5=귀속) ⇒ 경계 폐쇄(N3=폐쇄) | 태어나야/지정되어야 들어가는 집단은 정의상 가입 불가 | 폐쇄라고 귀속은 아님 (비공개 동호회는 성취·폐쇄) |
| F7 | unfocused(N2) ⇒ 공동 **상호작용** 과업 없음 | 공동 초점이 없으면 함께 수행하는 과업도 없음. 단 개인 목적은 각자 있음(버스: 각자 이동 목적). ※ 매핑 검증에서 예외 유형 발견: **집계형(aggregative) 행동** — 선거·크라우드펀딩은 상호작용 없이 개인 행동의 합산으로 집합 결과를 만든다. unfocused이면서 집합적 산출이 있는 제3 범주 | 목적 없어도 focused 가능 (수다) |

이 규칙들은 04 매핑 매트릭스의 **일관성 검사기**로 쓴다: 매핑 결과가 F1~F7을 위반하면 판정 오류이거나 축 정의가 덜 다듬어진 것.

## 3. 상관 — 독립이지만 같이 움직이는 쌍 6개

| # | 상관 쌍 | 경향 | 반례 (그래서 독립) |
|---|---|---|---|
| C1 | 규모 ↑ ↔ 방향성의 일방향화 | 인원이 커지면 N:N 대화가 유지 불가능 (Dunbar 한계) → 방송형으로 전이 | 대규모 다방향: 축제·시위(군중이 상호작용), 온라인 커뮤니티 |
| C2 | 귀속(N5) ↔ CS 운영 논리(N8) | 가족·민족 등 귀속 집단은 공동 공유로 운영되는 경향 | 군대: 귀속(징집)인데 AR(서열)로 운영. 가족 상속 분쟁: 귀속인데 MP화 |
| C3 | 기한 없음 ↔ 관계지향(축11) | 끝이 정해지지 않은 집단은 관계 자체가 목적인 경향 (친구) | 회사: 기한 없는데 과업지향. 프로젝트 팀: 기한 있고 과업지향이지만 동창회는 기한 없고 관계지향 |
| C4 | 의례성(N11) ↑ ↔ 기록성(N12) ↑ | 의례는 방명록·명부·청첩장 등 기록 장치를 동반하는 경향 | 제사: 의례성 높지만 기록 없음. 계약: 의례성 낮지만 기록성 최강 |
| C5 | 반복성 ↑ ↔ 유대 강도 기여(N9) ↑ | 반복 노출이 유대를 강화 (Granovetter의 '함께 보낸 시간' 요소) | 매일 타는 같은 버스(반복·유대 0), 단 한 번의 결혼식(일회·유대 강화 큼) — **이 반례가 중요: 유대는 빈도가 아니라 의례성·정서 강도에서도 옴** |
| C6 〔v2 신설〕 | S2=비대칭 ↔ G4=AR 성분 | 위계 중심 구조(S2 비대칭)는 서열 운영논리(G4 AR)를 동반하는 경향 | **학급/학교**: S2=비대칭(담임)인데 G4=EM(또래 호혜). **가족**: G4에 AR 성분 있으나 S2=대칭. → 둘은 1:1 종속이 아니라 독립 차원(S2=중심/권위 구조, G4=멤버 간 장부)이므로 둘 다 유지. 이 반례가 "S2 비대칭을 G4로 흡수하지 않고 정식값으로 둔" 근거 (2026-06-15 데이터 검증) |

C5의 반례는 프로젝트 핵심 통찰과 직결된다: **신뢰 네트워크에서 이벤트의 가중치는 빈도 × 강도의 곱이고, 결혼식은 빈도 1에 강도 최대인 특이점**이다.

## 4. 직교 코어 축 확정

중복 제거(§1) 후 남은 축들을 적용 층위별로 나누고, 함의·상관이 약한 것들로 코어를 구성한다.

### 4-1. 이벤트 축 (1회 모임에 적용)

| 코어 축 | 값 |
|---|---|
| E1. 규모 | 2 / 3~15 / ~50 / ~150 / 150+ (Dunbar 구간) |
| E2. 초점성 (N2) | focused / unfocused |
| E3. 방향성 (축12) | 쌍방(1:1) / 방송(1:N) / 망(N:N) |
| E4. 매체 (축14) | 대면 / 동기 온라인 / 비동기 온라인 / 혼합 |
| E5. 의례성 (N11) | 통과의례 / 정기의례 / 비의례 |
| E6. 활동 모드 (N10) | 생성 / 선택 / 협상 / 실행 / 친교(과업 없음) |
| E7. 정서가 기본값 (축1) | + / 중립 / − (사회적 규범값, △주관) |

### 4-2. 집단 축 (지속 집단에 적용)

| 코어 축 | 값 |
|---|---|
| G1. 경계 (N3) | 개방 / 조건부 / 폐쇄 |
| G2. 멤버십 귀속성 (N5) | 귀속 / 성취 |
| G3. 동질성 기반 (N7) | 혈연 / 지연 / 학연 / 직연 / 취향 / 신앙 / 상황 / 없음 / 이해관계 〔v2: +신앙·이해관계〕 |
| G4. 관계 운영 논리 (N8) | CS / AR / EM / MP (복수 가능) |
| G5. 기한 (축10) | 명시적 기한 / 무기한 / 갱신형 〔v2: +갱신형〕 |
| G6. 소집 반복성 (축9) | 일회성(집단 아님) / 정기 / 비정기 / 상시 |

### 4-3. 공용 축 (이벤트·집단 양쪽 적용)

| 코어 축 | 값 |
|---|---|
| S1. 참여 강제성 (N1) | 자발 / 의무 / 강제 / 귀속 〔v2: +귀속〕 |
| S2. 호스트 구조 (N4) | 대칭 / 호스트-게스트 / 공급자-소비자 / 비대칭(위계) 〔v2: 비대칭 정식화〕 |
| S3. 금전 흐름 (N6) | 없음 / 회비 / 일방 지불 / 부조 / 쌍방 교환 / 역방향(임금) / 순환수령(ROSCA) 〔v2: +순환수령〕 |
| S4. 기록성 (N12) | 무기록 / 사적 기록 / 공식 기록 / 법적 기록 |

> **축 버전: 17축 v2** (2026-06-15). v1 대비 값 목록 확장(축 개수 17 불변): S1·S2·S3·G3·G5. 변경 근거·셀 영향은 [AXES-V2-CHANGELOG.md](./AXES-V2-CHANGELOG.md). 신설값(귀속·갱신형·신앙·이해관계·순환수령)과 S2 비대칭 정식화는 624셀 검증에서 확정된 관찰의 정식 반영이며 셀 판정은 계모임 S3 1건(회비변종→순환수령)을 제외하고 불변.

### 4-4. 직교성 점검

코어 17축 중 완전 직교가 아닌 것은 함의 규칙(F1~F7)과 상관(C1~C5)으로 명시돼 있다. 완전 직교 행렬을 만들기보다 **"함의 규칙이 명시된 준직교 집합"**이 실용적이다 — 실세계 분류 체계에서 완전 직교는 달성 불가능하고(McGrath의 원형 모델도 인접 범주가 겹침), 함의 규칙은 오히려 검증 자산이 된다.

### 4-5. 버린 축

| 버린 축 | 사유 |
|---|---|
| 과거/다가올 (축6) | 모임 속성이 아니라 관찰 시점 인덱스 (C층위) → 데이터 모델의 timestamp |
| 짧은/긴 (축5) | 단독 축으로는 정보량 낮음. 세션 길이는 이벤트 메타데이터(분 단위 수치)로, 집단 수명은 G5로 분해 흡수 |
| 기쁜/슬픈, 가고 싶은/싫은 원본 | B층위로 강등·보존 (E7에 사회 규범값만 남김) |
| 모임장 유/무 (축8) | S2(호스트 구조)와 E3(방향성)의 조합으로 거의 복원 가능. 리더십은 집단 내부 구조 연구(AR의 세부)로 미룸 |

## 5. 쪼개기/합치기 최종 요약

```
합침:  축3(유무료) ─┐
                    ├─→ S3 금전 흐름 (v2: 7값, +순환수령)
       (임금 발견) ─┘
       축4(혼성동성) ──→ G3 동질성 (v2: 9값, +신앙·이해관계)
       축13(의도) ──→ E2 초점성 (Goffman)
       축2(가고싶음) ──→ S1 강제성 + B층위 보존

쪼갬:  축5(짧은긴) ──→ 이벤트 세션 길이(수치) + G5 집단 기한
       축11(목적) ──→ E6 활동 모드(과업 5종) + C3 관계지향 경향
       축8(모임장) ──→ S2 호스트 구조 + (보류) 리더십 형태

승격:  "모임 vs 집단 vs 장소 vs 관계" 층위 구분 자체가 최상위 분류가 됨
```
