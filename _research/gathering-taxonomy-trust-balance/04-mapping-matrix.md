## 3. △(주관 판정) 셀의 근거

| 셀 | 쟁점 | 채택 근거 |
|---|---|---|
| 결혼식 E3=방송△ | 식순은 1:N 방송이지만 피로연은 N:N 망 | 의례 본체 기준으로 방송, 실제로는 **시간에 따라 방송→망으로 전이하는 복합 이벤트** — 단일값 강제가 부정확함을 기록 |
| 결혼식 S1=의무△ | 가까운 사이는 자발, 먼 사이는 의무 | **관계 거리의 함수** — B층위(참가자별)가 새는 대표 사례. 부조 액수가 이 거리의 정량 신호라는 점이 프로젝트 핵심 |
| 명절 E7=+△ | 규범은 +, 실제는 갈등 빈발 | 사회 규범값(+)을 적되 분산이 큰 축임을 표시 |
| 종교 예배 S1=자발△ | 신자에게는 내적 의무 | 외부 강제 없음 기준으로 자발 |
| 축제 E2=U△ | 군중 전체는 unfocused, 무대 앞은 focused | **공간적으로 F/U가 공존** — 축제는 unfocused 장 안에 focused 점들이 떠 있는 구조 |
| 가족 S1=귀속△ | 자발/의무/강제 어느 값도 안 맞음 | S1 값 체계의 빈틈 발견 — "귀속(선택 부재)"을 4번째 값으로 추가 제안 |
| 계모임 G5=기한△ | 곗바퀴 완주가 기한이지만 보통 재시작 | 갱신형 기한(renewable term)이라는 중간값 필요 |
| 종교 공동체 G3=신앙△ | 혈연~취향 어디에도 안 맞음 | G3 값 목록에 "신념" 추가 제안 |
| 크라우드펀딩 S3=부조△ | 리워드형은 일방 구매, 후원형은 부조 | 부조와 구매의 중간 — **부조의 디지털 연속 스펙트럼 증거** |

매핑 과정에서 값 체계의 빈틈 3개가 드러남 (S1 "귀속", G5 "갱신형", G3 "신념"). 분류 체계는 이렇게 사례가 역으로 축을 수정하게 두는 것이 건강하다.

### 결혼식이 특이점인 이유 (축 조합 증명)

매트릭스 전체에서 다음 5개 값이 **동시에 참인 행은 결혼식이 유일**하다:

```
E5=통과의례 ∧ E1=초대(전체 네트워크 소집) ∧ S3=부조(정량 신호) ∧ S4=공식 기록 ∧ 빈도=일생 1회
```

장례식이 가장 가깝지만 호스트 본인이 부재하고(상주가 대리), 돌잔치는 규모가 작다. 즉 "결혼식 = 신뢰 네트워크의 스냅샷"이라는 프로젝트 전제는 이 분류 체계 안에서 **축 조합의 유일성으로 형식화 가능**하다.
