# 결혼식 액션 카탈로그 — raw 동사 + project 해석 규칙 (결혼식)

> **두 층을 분리한다 (2026-06-15 정정).**
> - **raw 동사 (action_type)** = 사람이 한 *기계적 행위*. 해석값 없음. `action`에 저장. 예: 돈 건넴, 말함, 옴.
> - **해석 (부조·거래·축하…)** = project가 `(raw 동사 × event_type=결혼식 × 역할쌍)`으로 *계산*하는 출력(signal). 저장 안 함.
>
> 이 파일 = 결혼식 project 규칙표의 첫 판. §2 표 맨 왼쪽이 raw 동사, 나머지 칸은 project가 뱉을 값.
>
> **개정 (2026-06-16, Fiske 4모델 점검):** 단일 "신뢰이동 ○/△/✗" 칸은 *과부하*였다 — "2층 EM 잔액에 들어가나" 한 질문만 보느라 CS 유대도 MP 이행률도 ✗로 뭉갰다. 그래서 신뢰칸을 **EM·CS·신용 3층위로 분할**(아래 §2 범례). raw 동사에 CS·사전 신호(초대함·안아줌·동석) 보강.

## 1-2. 수집 가능성 (디방 입력 여부)

원칙: 개념 지도는 전부 그리되 **드롭하지 말고 ✓/△/✗ 태그만**. 필터(드롭)는 구현 단계에서.

**행위(raw 동사)별:**
| raw 동사 | 디방 수집 | 경로 |
|---|---|---|
| **초대함(청첩장)** | **✓** | **모바일청첩장 — 누구를 초대했나. 디방의 본질, 가장 직접적인 사전 관계 신호** |
| 돈 건넴(부조) | ✓ | 축의 QR |
| 서명함·말함(방명록 메시지) | ✓ | 디지털 방명록 |
| 글·영상 보냄(축전) | ✓ | 원격 방명록 메시지 |
| 옴(참석) | △ | RSVP·체크인 잡으면 |
| 물건 줌(화환) | △ | 주문·발주 연동 시 |
| 서비스 수행·돈 건넴(업체 거래) | △ | 유료 옵션 결제 기록 (→ 신용 3층 이행률) |
| 노래·말함(현장)·서약·선언·절·반지·박수·사진·대접·**안아줌·동석** | ✗ | 현장·구조화 안 됨 |

**역할별:**
| 역할 | 디방 수집 | 경로 |
|---|---|---|
| 신랑·신부 | ✓ | 청첩장 당사자 |
| 하객(+ 초대·부조·메시지 방향) | ✓ | 청첩장·방명록·축의 |
| 주례·사회자 | △ | 청첩장·식순에 적히면. **이 배정 자체가 신호**(05 H13) |
| 혼주·업체 | △ | 부분 (혼주=가족 연동, 업체=결제) |

→ **수집되는 신뢰 핵심(✓):** 초대 · 부조 · 방명록 메시지 · 축전 (+ 신랑/신부/하객 방향, 역할 배정). **2층 입력은 이걸로 출발**, ✗는 개념 지도로 남겨 나중에.

※ 신호원이 둘(05 H13): **행위 신호**(부조·방명록 — 주로 EM) + **역할·관계 신호**(누구를 초대·주례·사회자로 세웠나 — 주로 CS). 후자가 EM으로 안 잡히는 친밀을 직접 보완.

## 3. 핵심 관찰

- **신뢰칸이 과부하였다.** 단일 ○/△/✗로는 EM·CS·신용이 한 칸에 뭉개져, EM 렌즈가 CS(식사·동석·초대)와 MP(업체 이행률)를 ✗로 버렸다. 3층위 분할로 각자 제자리. → 이것이 "드롭 말고 층위별 태그"(수집가능 원칙과 동형).
- **"돈 건넴" 하나**가 역할쌍에 따라 부조(EM○)·공동부조(EM○)·거래(EM✗·신용○)로 갈린다 — 해석은 raw 동사가 아니라 `(동사 × 역할쌍 × event_type)`이 정한다.
- **초대·참석·축전·동석은 돈 0인데 CS○.** project가 돈뿐 아니라 시간·존재·인정·관계도 신호로 환산해야 함 — 결혼식=신뢰망 스냅샷의 핵심.

## 4. 층위별 핵심 (fold·신용이 실제로 쓸 것)

- **2층 EM(○):** 부조 · 참석 · 축전 · 서명 · 화환 · 공동부조 · 감사(사후)
- **2층 CS(○):** 참석 · 축전 · 초대(사전) · 동석·함께먹음 · 안아줌 · 축하인사 · 서약(부부) · 사진 · 축가
- **3층 신용(○):** 부조 이행 · **업체 거래 이행률(MP)** · (관찰: 약속·계약 이행)

## 5. 열린 항목

- **함께 울음·웃음(정서 공유)** — CS로는 맞지만 이산 액션으로 모호하고 수집 난망 → 별도 동사로 안 둠(보류).
- **AR(서열) 표현** — 좌석 배치·인사/축사 순서는 서열 신호지만 *액션이 아니라 구조*라 별도 판단 필요(현재 선언함만 AR).
- **비금전 동사 크기 수량화** · **부분청산 정도**(대접·답례품이 부조를 얼마나 상쇄) · **집단 동사 분해**(박수·공동부조).
- **한 액션이 두 신호** — 참석·축전·대접은 EM·CS 둘 다 ○. project가 한 행위에서 신호 둘을 뽑음(07 §5).
- **raw 동사 입도** — "말함"을 축하/감사/소개로 쪼갤지(역할쌍이 가르므로 보류).
