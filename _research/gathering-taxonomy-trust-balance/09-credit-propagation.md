# 09. 3층 신용 전파 — 그래프(엣지값) → 노드 신용 점수

> 2층(07·08)이 만든 가중 그래프를 **전파(Φ, PageRank류)**해 노드별 신용 점수를 낸다. "신용 = 네트워크 위치"(H12·요네다)의 구현 시제품. `sim.mjs`에 연결, Y10 상태로 실행.
>
> **성격:** 전파 연산자는 **PHI-5에서 유방향 정식 연산(Φ)으로 확정**(§1-2). default 페널티·합산 비중은 여전히 first-cut(§5, PHI-2/3 소관).

## 1. 파이프라인 (4단계)

자원 집계 원칙(엣지에서 이질 값 합치지 말고, 자원별 전파 후 점수 단계에서 목적별 합산):

```
2층 상태(이질 값 그래프)
 ① 자원별 동질 그래프 분리      → 부조 그래프 · CS 그래프   (대여는 sparse → 행동신호로)
 ② 자원별 유방향 전이(PHI-5): 부조=reversed-giving PageRank(기여자 적립)·CS=authority, d=.85 → 부조점수·CS점수 (0~1)
 ③ 행동 신호 직접 투입           → 이행점수 (대여 갚음=1.0 / default=0.2 / 무기록=0.7)
 ④ 점수 합산(용도별 비중)        → 최종신용 = 0.5·부조 + 0.3·CS + 0.2·이행
```

- **①②는 네트워크**(누구와 엮였나 → 전파). **③은 행동**(본인이 의무 지켰나 → 노드 직접, 전파 안 함).
- 환산 문제 회피: 같은 단위끼리만 전파, 합산은 0~1 점수의 가중합(이질 단위 아님).

**결정 3개 (first-cut, 임의·근사):**
| 결정 | 채택값 | 정밀화 후보 |
|---|---|---|
| 전파 방향 | ~~무방향~~ → **유방향·가중** (PHI-1 확정, §1-1) | 베푼 행위 기준·상대 신용 가중 재귀 전파 |
| default 투입 | 이행점수 **0.2**(고정) | 금액·비율·기간 비례, 데이터 학습 |
| 합산 비중 | 부조0.5·CS0.3·이행0.2 | 용도별(대출이면 이행↑) |

### 1-1. PHI-1 결정 (2026-06-17) — 전파 방향 확정 (위 표의 '무방향' 시제품을 대체)

09의 첫 시제품은 전파를 **무방향**(엮인 정도만)으로 근사했다. PHI-1에서 이를 **유방향·가중 전파**로 정식화한다. PHI-1의 범위는 "방향을 무엇으로 삼나"를 정하는 것이며, 구체 수식·수렴 검증은 PHI-5(전이/합성 규칙)와 별도 테스트로 넘긴다. 세 갈래의 방향을 못 박는다.

**① 엣지의 의미와 신용 적립 방향 = 베푼 행위 기준, 주는 쪽이 적립한다.**
EM 엣지는 "베푼/빌려준 행위(주는 사람 → 받는 사람)"로 읽는다. 신용은 그 행위를 한 쪽(주는 쪽)에 적립된다 — 신용을 "호혜를 지킨 행적"으로 보기 때문이다. 약속을 지켜 초대에 응해 참석한 것, 과거에 와준 것을 갚은 것, 아직 내 경조사는 없지만 미래를 위해 먼저 부조한 것이 모두 "행위자가 자기 호혜 계좌에 적립"으로 떨어진다(05 H12(4)의 "신용 = 본인 행적").
- **음수의 의미:** 이 계좌는 (준 것 − 받아 쓴 것)이라 음수가 될 수 있고, 음수는 "아직 적립보다 많이 당겨 쓴 상태"(예: 결혼식에서 미리 받아 씀)다. 이는 **기회 미도래** 마이너스이지 **떼먹은(default)** 마이너스가 아니다 — PHI-4의 "부조 미반환 = default 불가"와 같은 구조이며, 시간이 갚는다. (결혼 당사자가 평생 쌓은 신뢰를 결혼식에서 "사용"한다고 본 직관이 이 음수 허용 구조와 정합한다.)

**② 전파 = 상대의 신용으로 가중되는 재귀 전파.**
네 신용은 "얼마 베풀었나"만이 아니라 "누구에게 베풀었나(상대의 신용)"로 가중된다. 같은 부조 10만이라도, 갚을 가능성이 높고 신뢰망 중심인 사람에게 한 것이 갚을 일 없는 사람에게 한 것보다 신용을 더 올린다. 근거 둘: (i) 믿을 만한 상대는 호혜를 실제로 이어갈 확률이 높아 "죽은 베풂"이 아니고, (ii) 신용 있는 사람들과 주고받는다는 것은 네가 신뢰 클러스터 안에 있다는 신호다(05 H12(3) 고유벡터 중심성). 상대의 신용도 다시 그의 상대들로 정해지므로, 이 가중은 **재귀(고유벡터/부동점)**다.
- **방향의 미묘점:** 베푸는 *행위*는 A→B이지만, 신용 *영향*은 B의 신용이 A로 흐른다(상대 신용이 나를 끌어올림). 즉 행위 방향과 신용이 흐르는 방향이 반대다. 검색엔진에 비유하면 표준 PageRank(가리켜지는 쪽이 오름)의 거울 — "좋은 곳을 가리키는 허브" 점수에 가깝다.
- **왜 가중이 필수인가:** 모든 엣지를 동등(무가중)하게 두면 "건달/대부호"를 못 가린다(09 시제품 무방향의 한계). H12의 핵심 요구("신용 = 잔액 크기가 아니라 누구와 엮였나")를 살리려면 가중 전파가 필요하다. 덤으로, 가중 전파는 아무에게나 베풀어 점수를 올리는 악용도 억제한다 — 믿을 만한 상대와의 호혜만 실제로 값이 되기 때문.

**③ 리스크 전파 방향 = default한 사람 → 채권자 쪽 (기존 EM 노출 엣지를 탄다).**
PHI-4에서 온 결론. 이행 결과는 노드 신뢰도 `r(사람)`이고, default로 `r`이 낮아지면 그 낮음이 **이미 있는 EM 노출 엣지**를 따라 그에게 노출된 채권자에게 전파된다. 새 "이행 엣지"를 만들지 않는다. 방향은 노출의 방향 — 떼인 손실이 흐를 수 있는 통로(채권자)로 간다. (금전 노출이 없는 순수 평판 엮임에는 닿지 않으며, 그것이 연좌제를 피하는 올바른 동작임은 07 §1-1 (마) 참조.)

**PHI-1이 정하지 않는 것(다음 단계로 넘김).** 위 방향들을 *실제 수식*으로 만드는 일(가중 전파의 정확한 연산자·감쇠·수렴) = PHI-5(전이/합성 규칙). 음수·재귀에서 수렴하는지와 시뮬레이션에서 직관과 맞는지의 *검증* = 그다음 테스트. 자원 간 *합산 비중* = PHI-2. 노드 출력의 *해석* = PHI-3.

> **PHI-1 종료:** 전파는 유방향·가중. (① 주는 쪽 적립·음수=기회 미도래, ② 상대 신용으로 가중되는 재귀 전파·행위와 신용 영향은 반대 방향, ③ 리스크는 default→채권자 EM 노출 엣지.) 09 결정표의 "전파 방향 = 무방향"은 이로써 대체됨. → 다음: PHI-5(이 방향을 수식으로).

### 1-2. PHI-5 — 유방향 전이 연산자(Φ) 확정 (2026-06-17)

PHI-1 방향을 실제 수식으로 박았다. 도구는 EigenTrust 계보(04)에서 가져와 우리 베이스(부조 EM)에 맞춤.

**연산자(부조, 핵심) — reversed-giving PageRank:** "베푼 쪽이 적립, 신용 영향은 받은 쪽→베푼 쪽"(PHI-1).
```
recv[h] = Σ_g give[g][h]                                  (h가 받은 총 부조)
π_g = (1−d)·p_g + d·Σ_h ( give[g][h] / recv[h] )·π_h       (d=0.85, p=시드)
```
기여자 g의 신용 = 자기가 베푼 상대 h들의 신용을 *기여 몫(give[g][h]/recv[h])*만큼 물려받음. h의 신용은 다시 h의 상대들로 정해져 재귀(고유벡터/부동점). **받는 행위는 신용을 안 올리고 베푸는 행위가 올린다.**
- **CS(보조):** 유방향 authority PageRank(유대 받는 쪽 적립). **이행:** node 직접(L3-4).
- **정규화·음수(PHI-5.2):** 행을 recv[h]로 정규화(행확률). 음수(배신·default)는 전파행렬서 **제외(클리핑)** — default는 이행 node, 배신(CS<0)은 직접 페널티로. EigenTrust 클리핑 + PHI-4(이행=node)와 정합, 음수 전파 불안정 회피.
- **손잡이:** 감쇠 d=0.85 · 사전신뢰 시드 p(기본 균등) · correlation discounting(미적용, §5).

**검증(PHI-5.4, `sim.mjs`):** ① 부동점 수렴(부조 27회·CS 61회, tol 1e-13) ② 결합법칙=순서무관(TIMELINE 셔플 3종 동일) ③ 초기벡터 강건성(부동점 유일) ④ 직관(민수 최대 기여자→1위, 지훈 default 반영, 현우 고립 10위) — 모두 통과.

### 1-3. CAT-1 — 합성의 국소 정의(전이가중치 곱) + 카테고리 공리 (2026-06-17)

PHI-5(§1-2)는 *전역 전파*(부동점)만 정의했다. CAT-1은 그 밑에 깔린 **국소 합성 규칙**(엣지 둘을 이어 간접 신뢰를 만드는 한 칸 연산)을 명시하고, 그것이 카테고리 공리를 만족함을 검증한다.

**(가) 두 대수의 구분 — 핵심.** 우리 모델에는 *층이 다른* 두 연산이 있고 섞으면 안 된다.
- **2층 fold 대수:** 엣지값(부조 잔액)을 *쌓는* 연산 = (ℤ,+) 아벨군(신호 더하기, PHI-4). "같은 칸에 신호를 누적".
- **3층 경로 합성 대수:** 엣지값을 *이어* 간접 신뢰를 만드는 연산 = 곱(×). "한 칸 → 다음 칸으로 신뢰를 전달".

두 대수를 잇는 다리가 **PHI-5.2 정규화**다: 원장 잔액(ℤ) → 전이가중치([0,1]).

**(나) 전이가중치 w(g,h).** 정규화로 얻는 한 칸 값:
```
w(g,h) = give[g][h] / recv[h]        (recv[h] = Σ_g give[g][h] = h가 받은 총 부조)
```
의미 = "h가 받은 부조 중 g가 댄 몫" = h의 신용이 g에게 흘러갈 비율(PHI-1의 '신용 영향: 받은 쪽 h → 베푼 쪽 g'). 이 w들이 전이행렬 M의 성분이며 **열-확률**이다(받은 적 있는 h의 열 합=1 — 신용이 새지 않음).

**(다) 국소 합성 규칙 — CAT-1 산출물.**
```
(g ← i, 중간자 h 경유) = w(g,h) · w(h,i)                       ← 두 칸을 '곱'으로 이음
g의 i에 대한 2-경로 간접 신뢰 = Σ_h w(g,h)·w(h,i) = (M²)[g][i]   ← 중간자에 대해 '합'
k-경로 = (Mᵏ)[g][i] ;  직접 포함 전체(감쇠 d) = (1−d)·Σ_{k≥0} dᵏ (Mᵏ)[g][i]
```
- **반환(semiring) 구조:** 경로를 *잇을* 땐 ×(곱), 여러 경로를 *합칠* 땐 +(합). 즉 〈전이가중치, 경로연결=×, 경로합산=+〉. 베이스 단위 = 1.
- **항등원(CAT-2 미리):** M⁰ = I(자기신뢰 = 1)가 합성의 단위.

**(라) 전역 전파와의 일치(CAT-4 미리).** 위 '모든 경로의 감쇠합'이 곧 PHI-5 전파 부동점이다:
```
π = (1−d)p + d·M·π     ⟺     π = (1−d)·Σ_{k≥0} dᵏ Mᵏ p
```
좌변(반복 부동점) = 우변(직접 k=0 + 한다리 k=1 + 두다리 k=2 … 의 감쇠합). 즉 **"전파" = "모든 간접 경로 합성의 합"**.

**(마) 검증 결과 (`sim-cat1.mjs`, Y10):**

| # | 항목 | 결과 |
|---|---|---|
| 1 | M 열-확률(신용 보존) | ✅ 받은 노드 열 합 = 1 |
| 2 | 2-경로 합성 = 중간자별 곱의 합 | ✅ 워크드 예 — 민수←영희: 0.313·0.192 + 0.385·0.192 = **0.1341** = (M²)[민수][영희] (중간자 수진·지훈) |
| 3 | 전파 부동점 = 경로 합성 감쇠합 | ✅ 오차 1.1e-15 |
| 4 | 결합법칙 (MM)M = M(MM) | ✅ 오차 2.8e-17 (합성=행렬곱이라 자동) |
| 5 | 항등원 I·M = M = M·I | ✅ 오차 0 |

**(바) 경계·정직 단서.**
- CAT-1의 범위 = *국소 규칙 명시 + 2-경로 수치 검증*. CAT-2(항등)·CAT-3(결합법칙)·CAT-4(전파=경로합성)의 *형식 진술*은 다음 칸 — 단 위에서 보듯 **셋 다 수치로는 이미 통과**(합성을 행렬곱으로 놓으면 결합·항등이 공짜, 부동점=경로합성합이 1e-15로 성립).
- **dangling 열**(받은 적 없는 노드, 예: 철수의 *부조* 열 합=0)은 전파에서 teleport로 처리. 순수 항등식(검증 3)은 dangling 재분배 없는 연산자로 확인했고, `sim.mjs propGiver`의 dang→seed 재분배는 질량보존용 동치 변형이다.
- "곱으로 경로 합성"은 표준적이되 **모델 가정**(링크 신뢰가 독립적으로 곱해진다)임을 유지한다.

## 3. 해석·검증

〔⚠️ PHI-5 유방향에선 순위가 바뀜 — 민수 1위·영희 4위. 아래 해석은 구 무방향 기준이며, 유방향 해석은 §1-2 참조.〕

- **영희 1위** — 본인 결혼식(Y5)에 부조가 크게 모여 부조망 중심(부조 1.00).
- **민수 2위** — 정기모임·여러 결혼식 참석으로 CS 허브(CS 1.00).
- **태호 4위** — 부조 약함(0.31)인데도 **대여를 갚아(이행 1.00)** 끌어올려짐.
- **지훈 7위** — CS 0.85로 높은데 **default(이행 0.20)가 끌어내림.** 만약 갚았다면(이행 1.0) 최종 ≈0.72로 3위권. **default가 ~4순위 비용.** 단 *밑바닥은 아님* — CS 망에 깊이 박혀 있어서. (현실적 — 한 번 default해도 잘 엮인 사람은 안 무너짐.)
- **현우 10위** — 태호 1명에만 연결(부조 0.08·고립) → 망 주변부.

검증: 망 중심(영희·민수)↑·고립(현우)↓·이행 좋음(태호)↑·default(지훈)↓ — 전부 직관과 설명 가능. "엣지값(2층) → 노드값(3층)"이 떨어지고, 행동 신호가 직접 반영됨.

## 4. 일관성 (H12·07·08과)

- **H12** — "신용 = 네트워크 위치(페이지랭크), 잔액 크기·부조 이행률이 아님" ✅. 부조는 *엣지(전파)*로, 대여 이행은 *행동 직접*으로 — 경로 분리(H12 보강2) 그대로.
- **07** — 입력은 fold 상태(EM·CS 원장). default판정 키로 분리된 대여가 이행점수로 감.
- **08** — 같은 시뮬 데이터. 08의 거친 미리보기(×0.5 페널티)를 본 파이프라인(이행점수 차원)으로 교체·정식화.
