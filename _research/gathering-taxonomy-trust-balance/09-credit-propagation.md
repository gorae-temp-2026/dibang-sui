# 09. 3층 신용 전파 — 그래프(엣지값) → 노드 신용 점수

> 2층(07·08)이 만든 가중 그래프를 **전파(Φ, PageRank류)**해 노드별 신용 점수를 낸다. "신용 = 네트워크 위치"(H12·요네다)의 구현 시제품. `sim.mjs`에 연결, Y10 상태로 실행.
>
> **성격:** first-cut 시제품. 결정값(무방향·default 페널티·합산 비중)은 임의·근사 — 정식은 데이터 보정 필요(§5).

## 1. 파이프라인 (4단계)

자원 집계 원칙(엣지에서 이질 값 합치지 말고, 자원별 전파 후 점수 단계에서 목적별 합산):

```
2층 상태(이질 값 그래프)
 ① 자원별 동질 그래프 분리      → 부조 그래프 · CS 그래프   (대여는 sparse → 행동신호로)
 ② 자원별 PageRank(무방향,d=.85) → 부조점수 · CS점수 (0~1)
 ③ 행동 신호 직접 투입           → 이행점수 (대여 갚음=1.0 / default=0.2 / 무기록=0.7)
 ④ 점수 합산(용도별 비중)        → 최종신용 = 0.5·부조 + 0.3·CS + 0.2·이행
```

- **①②는 네트워크**(누구와 엮였나 → 전파). **③은 행동**(본인이 의무 지켰나 → 노드 직접, 전파 안 함).
- 환산 문제 회피: 같은 단위끼리만 전파, 합산은 0~1 점수의 가중합(이질 단위 아님).

**결정 3개 (first-cut, 임의·근사):**
| 결정 | 채택값 | 정밀화 후보 |
|---|---|---|
| 전파 방향 | **무방향**(엮인 정도만) | 유방향(누가 누구를 믿나 — 더 정확) |
| default 투입 | 이행점수 **0.2**(고정) | 금액·비율·기간 비례, 데이터 학습 |
| 합산 비중 | 부조0.5·CS0.3·이행0.2 | 용도별(대출이면 이행↑) |

## 3. 해석·검증

- **영희 1위** — 본인 결혼식(Y5)에 부조가 크게 모여 부조망 중심(부조 1.00).
- **민수 2위** — 정기모임·여러 결혼식 참석으로 CS 허브(CS 1.00).
- **태호 4위** — 부조 약함(0.31)인데도 **대여를 갚아(이행 1.00)** 끌어올려짐.
- **지훈 7위** — CS 0.85로 높은데 **default(이행 0.20)가 끌어내림.** 만약 갚았다면(이행 1.0) 최종 ≈0.72로 3위권. **default가 ~4순위 비용.** 단 *밑바닥은 아님* — CS 망에 깊이 박혀 있어서. (현실적 — 한 번 default해도 잘 엮인 사람은 안 무너짐.)
- **현우 10위** — 태호 1명에만 연결(부조 0.08·고립) → 망 주변부.

검증: 망 중심(영희·민수)↑·고립(현우)↓·이행 좋음(태호)↑·default(지훈)↓ — 전부 직관과 설명 가능. "엣지값(2층) → 노드값(3층)"이 떨어지고, 행동 신호가 직접 반영됨.

## 4. 일관성 (H12·07·08과)

- **H12** — "신용 = 네트워크 위치(페이지랭크), 잔액 크기·부조 이행률이 아님" ✅. 부조는 *엣지(전파)*로, 대여 이행은 *행동 직접*으로 — 경로 분리(H12 보강2) 그대로.
- **07** — 입력은 fold 상태(EM·CS 원장). default판정 키로 분리된 대여가 이행점수로 감.
- **08** — 같은 시뮬 데이터. 08의 거친 미리보기(×0.5 페널티)를 본 파이프라인(이행점수 차원)으로 교체·정식화.

## 5. 한계 · 다음

- **결정값 임의** — 무방향·default 0.2·비중 0.5/0.3/0.2 다 first-cut. 특히 default 영향(이행 가중 0.2)이 현실보다 약할 수 있음(실제 신용은 default를 크게 깎음) — 튜닝/데이터 보정 대상.
- **무방향 근사** — 신용은 본질적으로 방향성("누가 누구를 믿나"). 유방향 전파 + 방향 의미(받은 부조 vs 준 부조) 결정이 다음 정밀화.
- **자원 환산은 점수 단계로 미룸** — 엣지에서 안 합치고 0~1 점수 가중합으로 처리(성공). 단 비중은 용도별 정책으로 더 정해야.
- **실데이터 아님** — 08과 동일 한계(임의 수치).
- **hackathon 𝒲/Φ 연결** — 본 PageRank는 Φ(=S∘L)의 시제품. 정식 Φ·요네다(신용=Hom패턴)·MEC 정량모델은 그 트랙에서 이어감 → **3층 본진은 hackathon 트랙**, 본 문서는 2층과 잇는 다리·시제품.
