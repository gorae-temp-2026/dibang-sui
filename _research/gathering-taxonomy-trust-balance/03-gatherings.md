# 03. 모임 목록 정규화 — 층위 분리·중복 정리·누락 보강

> 선행 문서: [01-axes.md](./01-axes.md) §3 (이벤트/집단/장소/관계 분해), [02-axis-relations.md](./02-axis-relations.md)
> 원자료: 사용자 브레인스토밍 2번 관점 (~70개 항목)

---

## 1. 정규화 규칙

1. **층위 태그 부여**: 각 항목을 이벤트(E) / 집단(G) / 장소(P) / 관계(R)로 분류. 둘 이상이면 복수 태그.
2. **장소(P) 항목은 매핑 대상에서 제외**하되, 그 장소에서 발생하는 대표 이벤트로 변환해 보존 (예: 버스 → "대중교통 동승"이라는 unfocused 이벤트).
3. 유사 항목은 **묶음(family)**으로 통합하고 원본 명칭을 구성원으로 기록.
4. 이름은 "무엇이 일어나는가"가 드러나게 재명명 (예: "동물원" → "동물원 관람 동행").

| 학회/컨퍼런스 | 학회, 컨퍼런스 | 약한 유대(weak tie) 대량 생성 장치 |

### 2-2. 집단 (G) — 지속적 구성원 집합

| 정규화 이름 | 원본 항목 | 비고 |
|---|---|---|
| 가족 | 가족 모임(의 주체) | 귀속·CS의 원형 |
| 회사 | 회사, 공장 사무실 | AR+MP. "공장 사무실"은 장소지만 의도는 직장 집단으로 해석 |
| 친구 집단 | 친구 모임, 동네 친구 모임 | 무기한·대칭·CS/EM |
| 동창회 | 동창 모임 | 휴면-재활성형 집단 (기한 있던 집단의 사후 잔존) |
| 동아리/동호회 | 동아리, 동호회 | 취향 기반 성취 멤버십 |
| 학생회 | 학생회 | 임기형 기한 |
| 연구실 | 연구실 | 도제형 AR |
| 군대(부대) | 군대 | 강제·귀속(징집)·폐쇄의 극단 |
| 학급/학교 | 학교 | 기한 명시 집단의 원형 (사용자가 직접 든 예) |
| 온라인 커뮤니티 | 인터넷 커뮤니티, 온라인 | 익명·개방·무기록 신뢰의 문제 사례 |
| 선원 크루 | 원양 어선·유조선(의 승무원) | 항해 근무(E)의 주체 |

### 2-3. 장소 (P) — 매핑 제외, 이벤트로 변환됨

집, 독서실, 공원, 동물원, 버스, 택시, 비행기, 공장 사무실, 팝업 스토어, 학교(건물로서의), 카페. → §2-1의 "대중교통 동승", "시설 공존", "소매 거래" 등으로 흡수. **"집"만은 변환 불가** — 집에서 일어나는 모임은 가족 일상부터 집들이까지 다양해서 장소가 이벤트를 결정하지 않는 대표 사례.

### 2-4. 관계 (R) — 분류 대상이 아니라 결과물

친구, 연인(데이트의 누적), 임대인-임차인(계약의 지속 상태). → 신뢰 네트워크의 엣지. 이벤트의 누적이 만들고 갱신하는 대상이므로 별도 트랙.

## 3. 누락 보강 — 추가해야 할 모임 24개

선정 기준: (a) 한국인의 생애에서 보편적인데 빠진 것, (b) 신뢰·금융 모델에 구조적으로 새로운 정보를 주는 것, (c) 축 공간에서 기존 항목이 못 채우는 영역.

### 3-1. 의례 계열 (E)

| 추가 항목 | 추가 이유 |
|---|---|
| **제사/차례** | 의례성 최고인데 기록성 0, 참석자가 죽은 자와의 관계로 묶임 — 의례 축의 극단값 |
| **종교 예배 (교회/성당/절)** | **가장 큰 누락.** 정기 의례 + 헌금(독특한 금전 흐름) + 강한 공동체. 한국 신뢰 네트워크에서 교회의 비중은 막대함 |
| 졸업식/입학식 | 기한 있는 집단(학교)의 경계 이벤트 — 집단의 생성·소멸 순간 |
| 집들이 | 호스트-게스트 + 선물 — 결혼식의 미니어처 |
| 백일/기일 | 돌잔치·제사의 형제 항목 (생애 의례 주기 완성) |

### 3-2. 금융·호혜 계열 (E/G) — 프로젝트 핵심

| 추가 항목 | 추가 이유 |
|---|---|
| **계모임(契)** | **이 프로젝트에서 결혼식 다음으로 중요한 누락.** 신뢰 네트워크가 직접 금융 장치(곗돈)로 작동하는 한국 고유 사례. ROSCA(rotating savings)의 한국형. 기존 hackathon 리서치의 𝒲 모델이 이미 다루는 대상 |
| **품앗이** | 노동의 호혜 장부 — 부조의 노동 버전. EM의 원형 |
| 더치페이 회식 | EM(동등 맞춤)의 일상 사례. 계산 순간의 정산 구조가 N6 축의 또 다른 값 후보 |
| 주주총회 | 지분 비례 의결 — MP가 의사결정 구조까지 지배하는 사례 |
| 노동조합 | 갈등 모드 + 회비 + 연대 — 시위와 회사 사이의 빈 영역 |
| 크라우드펀딩 | 온라인 + 일방 지불 + 신뢰 선지급 — 부조의 디지털 변형 |
| DAO | 온체인 거버넌스 집단 — Sui 구현과 직결되는 참조점 |
