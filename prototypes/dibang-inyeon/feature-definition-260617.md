## 0. 정체성 / 위치

- **디방인연 = 데이팅앱(세로 피드)**. 디방 유니버스의 한 영역. **틴더식 좌우 스와이프가 아니라, 위로 스와이프=다음 사람**(블라인드/틱톡식). 이음은 신중한 행동이라 버튼으로 분리.
- **테마: 딥네이비/블랙 다크** — "유니버스"라는 의미를 시각화. (다른 디방 탭은 라이트)
- **네비 2종:** ① 인연 내부 = **우측 세로 레일**(웨딩 라운지 vrail과 통일) — 유니버스(피드)·받은이음·채팅·프로필, 스크롤 시 숨김. ② 하단 = **메인 앱 네비**(Inyeon·Event·QR·My event·Setting). 이 파일은 인연 단독 목업이라 메인 네비는 자리표시.
- **월드맵은 인연이 아니라 Event list로 이동**(내가 초대/등록한 이벤트만). 라운지 내부(모이가모인곳)는 기존 유지.
- **모든 대화(DM)는 인연/유니버스 영역.** 라운지엔 이벤트 로그만 쌓임.

## 1. 기능 목록 (확정)

1. **세로 피드 탐색** — 위로 스와이프(또는 ✕ 넘기기 버튼) = 다음 사람. 아래로는 동작 없음.
2. **사진 보기** — 카드 **좌우 탭 또는 좌우 스와이프**로 갤러리 넘김(상단 세그먼트 점). 대표 포함 **앞 3장 무료, 4장째부터 잠금**(흐릿 + 중앙 "사진 더보기 🪙20" 오버레이) → §2·§3.
3. **이음 신청** — **노드–노드(점선) 아이콘 버튼**. 폼은 **한마디(주관식)만** + 미리보기(내 대표사진·이름·한마디). 이름은 구글 로그인 자동(객관식 관계·나이·성별 입력 없음). 신청 자체 무료.
4. **프로필 상세** — 카드 탭 또는 하단 "프로필 보기 ▾" → **아래에서 올라오는 한 페이지(분할 없이 통스크롤)**: 소개·출처(어디서 마주쳤나)·신뢰네트워크(익명 분자 그래프)·신뢰잔액(익명 범위)·공통친구. **사진 탭하면 닫힘**. 하단 이음 신청 버튼 고정. 이름·하객·네트워크명은 이음 후.
5. **매칭 범위(관계 거리)** — ⚙ → **가로 게이지 슬라이더(1다리 ~ 6다리 이상, 양손잡이)**로 볼 범위 지정. 기본 랜덤, **보는 건 무료**. (거리/지역 필터 없음 — 물리거리 대신 관계거리가 축)
6. **관심 표현 = 추가 사진 열람**(요네). 별도 "좋아요/하트"는 **없앰** — 사진을 열어 보는 행위 자체가 관심 신호.
7. **이음 = 버튼**(폼) → 보내면 "내가 보낸 이음(대기)" → **상대 수락 시 매칭**. 채팅은 **양방향 이음**에서만, 대화 여는 시점에 거리별 요네 차감(§2).
8. **받은이음 탭** — 세그먼트 [이음 | 관심]. **이음** = ⓐ 나에게 온 이음 신청(수락/거절) + ⓑ 내가 보낸 이음(대기) / **관심** = ⓒ 나를 본 모이(내 사진 본) + ⓓ 내가 본 모이(내가 연).
9. **채팅** — 상단 **메모리**(이음된 모이가 올린 2초 영상, 카드형 뷰어) + 대화 목록.
10. **내 프로필** — 대표사진(디방 공통, **Setting에서만 변경**) + 인연 추가사진 슬롯 + 인연 소개글 + 신뢰잔액(Moi Credit 재료) → §3.
11. **신고 · 차단** — 프로필 상세에서. 차단 시 서로 추천에서 제외.

## 2. 요네 이코노미 (핵심)

> **요네의 의미 = "관계를 잇고 대화를 여는 데 드는 에너지."** 아는 사람(같은 이벤트)과는 공짜, 모를수록 든다.
> 요네는 디방 전체 화폐(SUI/USDC 충전). **충전은 메인 네비 Setting에서.** 인연 안에선 부족 시 충전 넛지만.

| 행위 | 요네 | 비고 |
|---|---|---|
| 보기 · 매칭범위(관계거리) 필터 | **무료** | 누구를 볼지 고르는 것은 비용 없음 |
| 추가 사진 열람 | **요네**(데모 20) | **대표 포함 3장 무료, 4장째부터 요네.** 열람 = 상대에게 "누군가 관심" 알림(익명) |
| 이음 신청 보내기 | **무료** | 내 이름·관계를 먼저 공개 |
| 상대 수락 → 이음 성사(매칭·네트워크 공개) | **무료** | 사회적 관계는 무료로 성립 |
| **대화 열기** | **거리별 차감** | 같은 이벤트 **0** / 2~3다리 **50** / 모르는 사람 **200** (데모값) |

**차감 시점 = "대화 여는 순간"(상대 수락 후).** 매칭(관계)은 성립하되, **그 사람과의 채팅 채널을 여는 데만** 요네가 든다.

**부담 주체:** **먼저 다가간 쪽(이음 신청을 보낸 쪽)이 대화 열기 요네를 낸다.** → 내가 받은 이음 신청을 수락하면, 비용은 상대가 부담하므로 나는 바로 대화가 열린다.

**요네 부족 처리:** 상대가 이음을 수락했어도 내 요네가 모자라면 대화가 안 열림. 채팅 목록/매칭 화면에 *"이음 수락됨 · 요네 부족 — 충전하면 바로 열려요"* 잠금 카드 + 안내. 충전 시 즉시 열림.

## 3. 프로필 사진 정책

- **대표 사진 = 디방 전체 공통(모이 프로필)**. 정체성 일관 = 신뢰 강화. Setting에서 설정.
- **추가 사진 = 인연 전용 갤러리** — 인연 탭 내 프로필에서 여러 장 업로드. **대표 포함 앞 3장은 무료 노출, 4장째부터 열람 시 요네**(= 관심 신호).
- **노출 토글(중요):** "나를 인연 매칭 풀에 넣을지" on/off. 결혼식 하객으로만 쓰고 데이팅은 원치 않는 사람 보호. → Setting/프로필.
- (구현 위치) 대표 프로필·노출 토글 = Setting / 추가 사진·소개 편집 = 인연 탭.

## 4. 이음 · 대화 상태 흐름

```
[익명 카드 열람] (무료)
   ├─ 추가 사진 열람(요네) ──→ 상대에게 "관심" 신호(익명)
   ├─ 넘기기
   └─ 이음 신청(버튼·무료, 내 이름+한마디 먼저 공개)
        └─ [내가 보낸 이음 · 대기] → 상대 수락
              └─ [이음 성사] 이름·공통친구 공개 (전체 네트워크는 오프라인 만남 후)
                    └─ 대화 열기 (거리별 요네 차감, 먼저 다가간 쪽 부담)
                          ├─ 요네 충분 → [채팅 오픈]
                          └─ 요네 부족 → 잠금 + 충전 안내
```

받는 쪽: **받은 이음 신청 → 수락/거절.** 수락 시 매칭, 대화는 상대(신청자) 비용으로 바로 열림.

## 5. 익명 / 공개 규칙 (디방 동결)

- **이음 전(익명):** 보임 = 대표 사진 · 신뢰네트워크/신뢰잔액(익명 범위) · 누구와 이음됐는지(익명 노드) · **관계 거리(만난 적 있는 / 알 수도 있는 / 낯선)**. **숨김 = 이름 · 누구의 하객 · 어떤 네트워크 · 구체 이벤트명.**
  - ※ **나이는 디방에 개념 없음**(구글 로그인만, 입력 안 받음) — 이음 후에도 표시 안 함.
  - ※ 인연 첫 카드 헤드라인은 구체 식장명 대신 **generic**: "함께 참여한 결혼식이 있어요" / "두 다리 건너 아는 사이예요" / "아직 마주친 적 없는 새 인연이에요". (구체 식장·하객은 이음 후 대화에서)
  - ※ 내 프로필에 **인연 전용 소개글**(짧은 bio, 디방 전체 프로필과 별개) 입력 — 다른 모이가 내 프로필을 볼 때 노출.
- 이음 신청 = 내 관계+이름을 **먼저** 공개 → 상대 수락 시 상호 공개.

### 5.1 온라인 이음 vs 오프라인 만남 — 단계적 공개 (확정)
디방인연 매칭은 **온라인**이라 "실제로 만난 것"이 아니다. 소속·중심 네트워크는 **검증된 오프라인 co-presence**가 있어야 공개한다.

1. **이음 전:** 익명(사진·관계 거리·신뢰네트워크/잔액 익명·누구와 이음됐는지 익명).
2. **온라인 이음(인연 매칭):** **이름 + 본인이 밝힌 관계(주관식) + 공통으로 아는 사람(mutual) 이름** + 대화. → 그 외 **소속(누구의 하객)·전체 중심 네트워크는 비공개.** (공통 연결은 내가 이미 아는 사람이라 노출 부담 낮고 강력한 신뢰 신호 — LinkedIn mutual식. 이음 전엔 "공통 N명" 카운트만.)
3. **오프라인에서 실제로 만남(이음 후 같은 이벤트 재회 등):** 비로소 **소속 · 객관식 관계 · 중심 네트워크 전체 공개** + 같은 네트워크로 연결. (= 기존 결혼식 이음 완료 규칙)

→ 신뢰그래프 무결성 + **오프라인에서 만날 동기**(디방의 핵심)를 만든다.

## 6. Moi Credit 연결 (해커톤 심장)

- 이음·대화·관심(사진 열람)·이벤트 참여가 **신뢰 attestation**으로 적립 → **신뢰잔액** 상승 → **Moi Credit(온체인 신용)의 재료**.
- 인연의 매 행동이 곧 `event`→`project`로 들어가는 **action 로그의 원천**. (별도 산출 로직 명세 예정)

## 7. 세계관 어휘 (동결)

모이(나) · 모이가 모인곳(라운지 시각화) · 이음(1촌 맺기) · 요네(화폐) · 사이(관계) · 우리의 온기(network contribution).

## 8. 미해결 / 추후

- 추가 사진 열람 요네값·대화 열기 요네값 = **데모 임시값(20 / 0·50·200)**, 무료 사진 = **대표 포함 3장**. 실서비스 단가·장수는 별도 확정.
- 양쪽 다 대화 비용 부담 모델은 비채택(먼저 다가간 쪽 부담).
- **제스처(확정):** 위 스와이프=다음 사람 / 좌우=사진(탭·스와이프) / 아래=없음 / 카드 탭·하단 "프로필 보기 ▾"=프로필 상세. 둘러보기 그리드는 제거.
- **신뢰잔액 효용:** 인연 내 매칭 흐름에 직접 영향 줄지(우선 노출 등)는 보류 — Moi Credit 뒷단 연결점이라 추후 결정.
- **성별/지향 필터:** 현재 미고려(개방형).
- 기술: Tailwind + shadcn/ui + React + TS, 직접구현 금지·검증 라이브러리, 실제 코드는 Claude Code. (분자 네트워크 = react-force-graph/d3-force 후보 — 합동 세션 확정)
