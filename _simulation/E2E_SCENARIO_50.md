# E2E 시나리오 — 50 지갑 + 사용자 계정 (v3)

> v2 → v3: 버그 발견 시 즉시 수정 후 이어서 진행, 프로필 사진 설정 Phase 추가, 프로필 이미지 소스 지정

## 실행 방식

- **기본**: Playwright UI 경유 (dev 로그인)
- **로그인**: `sessionStorage.setItem('dibang.dev.sk', sk)` → 새로고침 → 자동 로그인 (Ed25519 직접 서명)
- **로그아웃**: `sessionStorage.removeItem('dibang.dev.sk')` → 새로고침

### 버그 발견 시 처리 흐름

```
1. Bug Report 메모에 기록 (아래 양식)
2. 즉시 bug fix 태스크 생성
3. 코드 수정 + tsc/빌드 확인
4. 커밋
5. E2E 테스트 이어서 진행 (중단점부터 재개)
```

> ~~SDK 스크립트로 우회~~는 하지 않는다. 버그를 발견하면 **고치고** 이어서 진행.

### 사용자 계정(0x46e7) 처리

사용자 주소는 zkLogin이라 dev 로그인(Ed25519)으로 접근 불가.
→ **사용자 계정 관련 액션은 SDK 스크립트로 실행** (이건 버그가 아니라 구조적 제약).
→ 시나리오에서 사용자 역할은 "50개 지갑이 사용자에게 이음 신청 / 결혼식 초대"하는 방향으로만 자동화.

### 프로필 사진 소스

일부 계정은 실제 프로필 사진을 사용:
- 경로: `/prototypes/dibang-inyeon/profile-photos/`


## Bug Report + 즉시 수정 흐름

### 양식

```
### BUG-XXX: [기능명]
- **Phase/Step**: Phase X, Step Y
- **계정**: [이름] ([주소 앞 8자리])
- **시도**: [무엇을 했는지]
- **결과**: [무엇이 안 됐는지]
- **콘솔**: [에러 메시지]
- **수정**: [어떻게 고쳤는지 — 커밋 해시]
```

### 처리 절차

1. 버그 발견 → 위 양식으로 기록
2. **즉시** bug fix 태스크 생성 (TaskCreate)
3. 코드 수정 → tsc + 빌드 확인
4. 커밋 (bug report의 "수정" 필드에 커밋 해시 기록)
5. E2E 테스트 중단점부터 재개
6. 같은 기능 재테스트 → 통과 확인

> SDK 스크립트로 우회하지 않는다. 고치고 이어서 간다.
