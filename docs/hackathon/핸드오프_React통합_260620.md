  4. **피드** — 기존 compose.
  5. **들러리·선물** — 네트워크 기여(§4).
- **디스플레이는 레일에서 제거** → **LIVE 축하메세지 카드 우측하단 `⛶ 전체보기`** 로 진입.

## 3. 모이가모인곳 (광장 plaza) — 신규 핵심 ★개념 정정 2026-06-20

> **정정:** "작은 미니룸"이 아니라 **45도 부감(俯瞰)의 넓은 광장**이다. **150~300명의 모이가 모이는 공간 = 라운지의 시각화.** 싸이월드 미니룸처럼 꾸미되, 스케일은 광장.

- **45도 위에서 본 넓은 광장.** 바닥(부감 평면) + 그 위에 **다수 모이(150~300) 스프라이트**가 모여 있음. 세로 풀스크린.
- **테마 스킨 = 데코 레이어 스왑.** 같은 광장 바닥·군중은 유지하고 **장식 세트만 교체**:
  - 결혼식 → 꽃아치 + 버진로드 + 분수 + 화분
  - 파티 → 풍선 + 테이블 + 가랜드
  - 클럽 → 무대 + 조명 + 댄스플로어
- **아트 디렉션 = 손그림(hand-drawn).** 기존 철수·영희 Excalidraw 손그림(검은 외곽선 + 플랫 컬러)을 **기본 모이**로 채택. 광장 바닥·데코·모이 변형도 **전부 같은 손그림 톤으로 통일**(반쪽만 손그림이면 미완성처럼 보임). *이전 "2.5D 아스가르드 chibi 렌더" 프레이밍은 이 손그림 노선으로 대체. 단 분수·꽃아치 모티프는 데코로 유지.* 레퍼런스: 업로드된 `brother-select_원본_전신.svg`·`sister-select_원본_전신.svg`, 톤 확인 목업 `해커톤/수이해커톤/(outputs) moi_plaza_wedding.png`.
- **렌더 = PixiJS(§7-1 확정).** 다수 스프라이트 + **팬/줌** + **모이 클릭 → 프로필 시트 → 이음**. 작게 들어가므로 디테일 캐릭터를 축소 사용. 프로필은 **디방인연 프로필과 동일**(§6, 통일).
- **에셋(손그림 톤, 별도 콘텐츠 트랙):** ① 광장 바닥 ② 테마별 데코 세트(결혼식/파티/클럽) ③ 모이 변형(색·옷). 상세 = `미니룸_에셋스펙_AI프롬프트_260620.md`(스케일/테마 반영해 갱신 필요).
- **샵(꾸미기)은 이 화면 안에서 진입** — 광장 데코·내 모이 옷을 요네로 꾸미는 맥락.
- **데모 광장 = 김수영·박소율 웨딩라운지(결혼식 테마).**
- 구현: **React + PixiJS.** 위험·신규부라 **먼저 placeholder 스텁(광장 바닥 + 컬러도형 군중 + 데코 1세트 + 팬/줌·클릭)** 로 시스템 완성 → 손그림 에셋 PNG 슬롯 교체.
